# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

後楽園ホールの座席表から席を選び、その席に座ったときのプロレス観戦視点を3Dで確認するビューア。Vite + TypeScript + Three.js、フレームワークなしの静的サイト。

## コマンド

```sh
npm run dev                      # 開発サーバー (http://localhost:5173)
npm test                         # vitest (src/**/*.test.ts)
npx vitest run -t 'parseSeatId'  # テスト名で絞り込み
npm run build                    # tsc --noEmit + vite build → dist/
npm run build:seats              # 座席データを公式PDFから再生成（要 pdftotext）
```

lint は入れていない（tsc の `strict` / `noUnusedLocals` が実質的なチェック）。

## 座席データは公式座席表から生成している

`src/data/seat-layout.generated.ts` は**自動生成ファイル**。手で編集しない。
東京ドームシティ公式の「後楽園ホールプロレスリング座席表」PDFを `tools/build-seat-layout.mjs`
が読み、印字された座席番号の**印字位置そのもの**を座席のワールド座標に変換している
（全1571席 / 58列）。座席の並びを直したくなったら、生成スクリプト側を直して再生成する。

- PDFの座標(pt) → メートルの換算は `PT_PER_M = 20.6`。リング（エプロン込み約7.8m四方）と
  パイプ椅子のピッチ(約0.45m)の2通りで出して一致することを確認した値。
- 東側・西側スタンド(A〜E列, 各109席)だけはPDFの文字レイヤーに入っていないため、
  図面から読み取った並びを `EW_STAND_COLUMNS` に定数として持っている。
- スクリプトはブロックごとの席数を検算して、合わなければ異常終了する。座席表PDFが
  差し替わって配置が変わったときはここで気付ける。

高さ方向はPDFにないので `src/data/hall.ts` の `RAKE`（各側の最前列の高さと傾斜）で与える。
公式サイトの「座席からの見え方」写真に合わせた近似値なので、見え方を調整するときはここを触る。
生成データは平面座標だけを持ち、高さは実行時に計算する——この分担を崩さないこと。

## アーキテクチャ

### 座標系と会場の非対称性

右手系 Y-up、単位はメートル、**リング中央が原点**。`+X`=東 / `+Z`=南 / `+Y`=上。
2D座席表は `x → SVGのx`、`z → SVGのy` に素直に落としている（＝公式座席表と同じく北が上）。

リングは会場の中央ではなく**北寄り**にある（南側スタンドが18列と深い）。`HALL` の範囲が
南北で非対称なのはそのため。中心対称を前提にしたコードを書かないこと。

### 座席の構造

10ブロック: 南側/北側/東側/西側スタンド、ステージ席(東)/(西)、四方のリングサイド。
`BlockSpec.kind` が `flat`（リングサイドのパイプ椅子、高さ0）か `stand`（ひな壇）かを持ち、
高さ・3Dの見た目・座席表の色分けがこれで決まる。ステージ席は北側スタンドと同じ斜面に乗るので、
`side` が `N` で、傾斜の起点も北側スタンドの最前列を共有する。

座席IDは `S-A-12`（ブロックコード-列-番）。列名はブロックによって英字(A〜R)・かな(い〜に)・
数字(ステージ席の1〜3)と異なる。`parseSeatId()` は「南側 H列 23番」「南H23」「N-K-45」
「ステージ席(東) 15番」などを受け付ける。**列名が数字のブロックがあるため、末尾の数字の
どこまでが座席番号かは決め打ちできない**——長い方から順に切って、実在する座席になった
組み合わせを採用している。

### 画面構成

`src/main.ts` が座席表パネルとビューアパネルの表示切替と、フォーム・ボタン(`data-action`)の配線を持つ。
3Dビューアは**最初に席が選ばれるまで生成しない**。座席表に戻るときは `viewer.stop()` で rAF を止める。

- `src/seatmap/seatmap.ts` — SVG座席表。拡大縮小・移動は `<g>` の transform だけで行い、
  1571個の `<rect>` は作り直さない。文字は `--map-scale` で拡大率を打ち消して大きさを保つ。
- `src/viewer/scene.ts` — ホールの手続き的3Dモデル。木のひな壇（北・東・西・ステージ席）、
  オレンジの固定席が並ぶ南側スタンド、リングサイドの赤いパイプ椅子、天井の照明トラス。
  座席は InstancedMesh でまとめて描く。ひな壇はベンチなので座席の形を持たない。
- `src/viewer/viewer.ts` — 一人称カメラ。カメラ位置は座席に固定し yaw/pitch を直接持つ
  （OrbitControls は使わない）。ドラッグ=見回し、ホイール=FOVズーム。

### ポインタ操作の注意

座席表・3Dビューとも `setPointerCapture` でドラッグを掴む。キャプチャ中は `click` イベントが
個々の `<rect>` ではなくSVGルートに飛ぶため、**座席の選択は click ではなく pointerup +
`document.elementFromPoint()` で解決している**。ここを `click` ハンドラに戻すと座席が選べなくなる。
SVGのテキスト要素は `pointer-events: none` にしておくこと（座席の上に重なってクリックを奪う）。

## 見え方を調整するときの参照先

公式サイトの座席表ページ（`/hall/seat/`）のカメラマークに、19か所の実際の見え方の写真がある
（`/hall/seat/modal/index_modal_0NN.html` の中の画像）。南側最前列/中段/最後列、北側最後列、
東西最後列などがあるので、対応する席で3Dを出して見比べると調整しやすい。
